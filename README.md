# 📱 NovaCell Support Chatbot — Multi-Source RAG Assistant

A grounded, retrieval-augmented (RAG) customer-care assistant for the fictional telecom
operator **NovaCell**. It answers Tier-1 support questions in plain language using *only*
NovaCell's own verified knowledge — FAQs, resolved tickets, official PDF guides, and the
plan & pricing catalog — and politely refuses anything out of scope.

Built with **LangChain + Chroma + Sentence-Transformers + Groq (Qwen3-32B)** and a
**Streamlit** chat UI.

---

## ✨ Features

- **Four knowledge sources**, each in its own vector collection:
  | Source | File | Collection |
  |---|---|---|
  | Public FAQ | `Data/faq.csv` | `faq` |
  | Resolved support tickets | `Data/tickets.db` | `tickets` |
  | Official PDF user guide | `Data/telecom_guide.pdf` | `guides` |
  | Plan & pricing catalog | `Data/plans.json` | `plans` |
- **Parallel retrieval** across all collections, merged into a single labeled context.
- **Strictly grounded answers** — never invents prices, plans, or policies.
- **Scope guard** — refuses trip planning, general knowledge, and personal account
  data (e.g. "what's my balance?") with a safe fallback message.
- **Streamlit chat UI** with one-click sample questions.

---

## 🏗️ Architecture

```
            INGEST                         RETRIEVE              ANSWER
faq.csv     ─► ingest_faq     ─► [faq]    ┐
tickets.db  ─► ingest_tickets ─► [tickets]│  RunnableParallel    Groq LLM
guide.pdf   ─► ingest_guides  ─► [guides] ├─► (top-k each) ─► ─► (grounded,
plans.json  ─► ingest_plans   ─► [plans]  ┘   merged context     scope-guarded)
                  │                                                   │
            MiniLM embeddings → Chroma (chroma_store/)          Streamlit app.py
```

The pattern for every source is identical: **load → build `Document` → embed → store in
its own Chroma collection → merge in the retriever.**

## 🚀 Quick start

### 1. Prerequisites
- Python 3.11+
- A free [Groq API key](https://console.groq.com)

### 2. Install
```bash
# with uv (recommended)
uv sync

# or with pip
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS/Linux
source .venv/bin/activate
pip install -e .
```

### 3. Configure your API key
```bash
cp .env.example .env
# then edit .env and set GROQ_API_KEY=your_real_key
```

### 4. Build the knowledge base (one-time, or after data changes)
```bash
python -m src.ingest_all
```
Expected output:
```
Ingested 25 FAQ entries.
Ingested 19 resolved tickets.
Ingested 37 guide chunks.
Ingested 14 plans.
```

### 5. Run

**Web app (Streamlit):**
```bash
streamlit run app.py
```
Open http://localhost:8501

**Terminal chat:**
```bash
python -m src.main
```

## 💬 Sample questions

**Plans & pricing** (grounded in `plans.json`):
- "What's your cheapest unlimited plan?"
- "Do you have a family plan?"
- "I'm going to Europe for about a week — what are my roaming options?"
- "Is there a discount for students?"

**Support** (FAQ / tickets / guide):
- "My mobile data is really slow. How do I fix it?"

**Out of scope** (safely refused):
- "What is my current bill balance?"
- "I am visiting London for 3 days in May, build me a trip plan."

## ⚙️ Configuration

Key settings live in [`src/config.py`](src/config.py):
- `EMBED_MODEL` — `sentence-transformers/all-MiniLM-L6-v2`
- `LLM_MODEL` — `qwen/qwen3-32b` (via Groq)
- `TOP_K` — documents retrieved per collection (default 3)
- `CHUNK_SIZE` / `CHUNK_OVERLAP` — PDF chunking
